# 炒股软件六大核心模块 API 接口设计规范

## 文档说明
1.  接口整体遵循 **RESTful 风格**，采用 HTTP/HTTPS 协议，编码格式统一为 UTF-8
2.  数据交互格式优先使用 JSON，响应体统一封装，错误码分层定义
3.  接口版本控制采用 URL 嵌入方式（如 `/api/v1/筹码分析/xxx`），便于后续迭代兼容
4.  所有接口均需进行身份认证（Token 放入 HTTP 请求头 `Authorization: Bearer {token}`）
5.  时间格式统一为 `yyyy-MM-dd HH:mm:ss`，价格单位为分（避免浮点精度问题，前端按需转换为元），分页参数统一标准化

## 一、 通用响应体格式
### 成功响应
```json
{
  "code": 200,
  "msg": "success",
  "data": {}, // 业务数据，按需返回对象/数组/基本类型
  "requestId": "req-1234567890abcdef", // 请求唯一标识，便于问题排查
  "timestamp": 1735689600000 // 响应时间戳（毫秒级）
}
```

### 失败响应
```json
{
  "code": 400/500, // 错误码，4xx 客户端错误，5xx 服务端错误
  "msg": "参数无效：股票代码格式错误", // 错误详情描述
  "requestId": "req-1234567890abcdef",
  "timestamp": 1735689600000
}
```

## 二、 通用分页参数（适用于多数据返回接口）
| 参数名 | 类型 | 是否必传 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| pageNum | Integer | 否 | 1 | 页码，从 1 开始 |
| pageSize | Integer | 否 | 20 | 每页条数，最大不超过 100 |

## 三、 六大模块接口详细设计

### 模块 1： 筹码分析接口

#### 1.1 获取单只股票筹码分布数据
- 接口路径：`/api/v1/chip/distribution`
- 请求方法：GET
- 接口描述：返回指定股票指定时间段内的筹码分布、集中度、主力成本等核心数据
- 请求参数（Query）：
  | 参数名 | 类型 | 是否必传 | 说明 |
  |--------|------|----------|------|
  | stockCode | String | 是 | 股票代码（如 SH600000、SZ000001） |
  | startDate | String | 否 | 开始日期（yyyy-MM-dd），默认近 30 天 |
  | endDate | String | 否 | 结束日期（yyyy-MM-dd），默认当天 |
- 响应数据（data 部分）：
```json
{
  "stockCode": "SH600000",
  "stockName": "浦发银行",
  "date": "2026-01-04",
  "chipDistribution": [ // 价格区间-筹码占比列表
    {
      "price": 850, // 价格（分）
      "chipRatio": 0.05, // 该价格区间筹码占比（0-1）
      "holderCount": 12000 // 该区间持仓户数（估算）
    }
  ],
  "chipConcentration": 0.89, // 筹码集中度（0-1，值越大越集中）
  "mainCostPrice": 820, // 主力成本价（分）
  "supportPrice": 790, // 筹码支撑价（分）
  "resistancePrice": 900, // 筹码压力价（分）
  "chipPeakInfo": { // 筹码峰核心信息
    "peakPrice": 830, // 主峰价格（分）
    "peakRatio": 0.65, // 主峰筹码占比（0-1）
    "isSinglePeak": true // 是否单峰密集
  }
}
```

#### 1.2 获取股票筹码趋势变化
- 接口路径：`/api/v1/chip/trend`
- 请求方法：GET
- 接口描述：返回指定股票多日筹码集中度、主力成本趋势数据
- 请求参数（Query）：
  | 参数名 | 类型 | 是否必传 | 说明 |
  |--------|------|----------|------|
  | stockCode | String | 是 | 股票代码 |
  | days | Integer | 否 | 统计天数，默认 60 天，最大 365 天 |
- 响应数据（data 部分）：包含多日趋势数组，结构略（复用 1.1 核心字段，按日期分组）

### 模块 2： 舆情分析接口

#### 2.1 获取单只股票舆情汇总
- 接口路径：`/api/v1/public/opinion/summary`
- 请求方法：GET
- 接口描述：返回指定股票舆情情绪评分、热点事件、舆情趋势摘要
- 请求参数（Query）：
  | 参数名 | 类型 | 是否必传 | 说明 |
  |--------|------|----------|------|
  | stockCode | String | 是 | 股票代码 |
  | timeRange | String | 否 | 时间范围（1d/3d/7d/30d），默认 7d |
- 响应数据（data 部分）：
```json
{
  "stockCode": "SH600000",
  "stockName": "浦发银行",
  "opinionScore": 75, // 舆情综合评分（0-100，越高越正面）
  "positiveRatio": 0.6, // 正面舆情占比
  "negativeRatio": 0.15, // 负面舆情占比
  "neutralRatio": 0.25, // 中性舆情占比
  "hotEvents": [ // 核心热点事件
    {
      "eventId": "event-123",
      "eventTitle": "浦发银行发布2025年年度财报，净利润同比增长5%",
      "eventTime": "2026-01-03 15:30:00",
      "eventInfluence": 80, // 事件影响力评分（0-100）
      "eventSentiment": "positive" // 事件情绪（positive/negative/neutral）
    }
  ],
  "opinionTrend": [ // 舆情评分趋势（按天/小时分组）
    {
      "time": "2026-01-01",
      "score": 72
    }
  ]
}
```

#### 2.2 获取股票舆情详情列表
- 接口路径：`/api/v1/public/opinion/list`
- 请求方法：GET
- 接口描述：分页返回指定股票的舆情原文、来源、情绪标签等详情
- 请求参数（Query）：包含通用分页参数 + stockCode + timeRange + sentimentType（舆情类型，可选）
- 响应数据（data 部分）：分页数据 + 舆情详情数组

### 模块 3： 风险控制分析接口

#### 3.1 获取账户风险评估报告
- 接口路径：`/api/v1/risk/account/assessment`
- 请求方法：GET
- 接口描述：返回当前登录账户的持仓风险、VaR、仓位合规性等评估数据
- 请求参数：无（基于登录账户身份获取数据）
- 响应数据（data 部分）：
```json
{
  "accountId": "acc-123456",
  "totalMarketValue": 10000000, // 账户总市值（分）
  "totalAvailableFunds": 2000000, // 可用资金（分）
  "varValue": 50000, // VaR值（95%置信区间，日最大潜在损失，分）
  "cvarValue": 80000, // 尾部风险值（分）
  "sharpRatio": 1.2, // 夏普比率
  "maxDrawdown": 0.08, // 最大回撤（0-1）
  "positionRiskList": [ // 单票持仓风险列表
    {
      "stockCode": "SH600000",
      "stockName": "浦发银行",
      "positionRatio": 0.3, // 持仓占比（0-1）
      "riskLevel": "medium", // 风险等级（low/medium/high）
      "stopLossPrice": 780, // 建议止损价（分）
      "stopProfitPrice": 950 // 建议止盈价（分）
    }
  ],
  "riskWarning": [] // 风险预警提示（如单票持仓超限）
}
```

#### 3.2 自定义止损/止盈规则配置
- 接口路径：`/api/v1/risk/stop/rule/config`
- 请求方法：POST
- 接口描述：创建/更新单只股票的止损/止盈规则
- 请求参数（Body）：
```json
{
  "stockCode": "SH600000",
  "stopType": "stopLoss", // stopLoss（止损）/stopProfit（止盈）
  "ruleType": "fixed", // fixed（固定比例）/moving（移动止损）/ma（均线止损）
  "value": 0.05, // 规则值（如固定比例5%传0.05）
  "isEnabled": true // 是否启用该规则
}
```
- 响应数据：操作结果确认

### 模块 4： 游资行为分析接口

  | 参数名 | 类型 | 是否必传 | 说明 |
  |--------|------|----------|------|
  | stockCode | String | 是 | 股票代码 |
  | startDate | String | 否 | 开始日期，默认近 7 天 |
  | endDate | String | 否 | 结束日期，默认当天 |
- 响应数据（data 部分）：
```json
{
  "stockCode": "SH600000",
  "stockName": "浦发银行",
  "totalNetBuy": 50000000, // 游资总净买入（分）
  "hotSeatList": [ // 核心游资席位列表
    {
      "seatId": "seat-123",
      "seatName": "XX证券上海溧阳路营业部",
      "seatType": "hotMoney", // hotMoney（游资）/institution（机构）
      "netBuy": 20000000, // 该席位净买入（分）
      "operationStyle": "shortTerm", // 操作风格（shortTerm/longTerm）
      "successRate": 0.65, // 该席位历史操作成功率
      "operationRecord": [ // 该席位近期操作记录
        {
          "tradeDate": "2026-01-03",
          "buyAmount": 30000000,
          "sellAmount": 10000000
        }
      ]
    }
  ]
}
```

#### 4.2 获取游资行为预警列表
- 接口路径：`/api/v1/heat/flow/alert/list`
- 请求方法：GET
- 接口描述：分页返回当前市场游资大额异动、联动操作等预警信息
- 请求参数：通用分页参数 + 预警等级（low/medium/high）
- 响应数据（data 部分）：分页数据 + 预警详情数组

### 模块 5： 大单异动分析接口

- 接口路径：`/api/v1/order/large/real-time`
- 请求方法：GET
- 接口描述：返回指定股票当日实时大单（特大单）成交、净流入、异动信号

  | 参数名 | 类型 | 是否必传 | 说明 |
  |--------|------|----------|------|
  | stockCode | String | 是 | 股票代码 |
  | largeOrderThreshold | Integer | 否 | 大单阈值（分），默认 5000000（50万） |
- 响应数据（data 部分）：
```json
{
  "stockCode": "SH600000",
  "stockName": "浦发银行",
  "currentPrice": 850, // 当前股价（分）
  "totalLargeOrderAmount": 100000000, // 当日大单总成交金额（分）

      "tradeTime": "2026-01-04 15:00:00",

      "tradeVolume": 10000, // 成交股数
      "tradeAmount": 8500000, // 成交金额（分）
      "tradeDirection": "buy" // 买卖方向（buy/sell）
    }
  ],
  "abnormalSignal": [ // 大单异动信号
    {
      "signalType": "continuousNetInflow",
      "signalTime": "2026-01-04 14:30:00",
      "signalDesc": "连续3笔大单净流入，异动强度高"
    }
  ]
}
```

#### 5.2 获取股票大单趋势数据
- 接口路径：`/api/v1/order/large/trend`
- 请求方法：GET
- 接口描述：返回指定股票大单净流入/流出的分时/日度趋势
- 请求参数（Query）：stockCode + timeType（分时/day，默认 day） + days（统计天数）
- 响应数据（data 部分）：趋势数组数据

## 四、 核心错误码定义
| 错误码 | 错误描述 | 所属场景 |
|--------|----------|----------|
| 200 | 操作成功 | 所有接口通用 |
| 400 | 参数无效/缺失 | 客户端请求参数错误 |
| 401 | 未授权/Token过期 | 身份认证失败 |
| 403 | 权限不足 | 无接口访问权限/账户被限制 |
| 404 | 资源不存在 | 股票代码无效/接口路径错误 |
| 429 | 请求过于频繁 | 接口限流触发 |
| 500 | 服务端内部错误 | 服务器逻辑异常/数据库错误 |
| 503 | 服务暂不可用 | API服务维护/第三方依赖故障 |
| 60001 | 股票代码格式错误 | 筹码/舆情/技术分析等模块 |
| 60002 | 账户不存在 | 风险控制模块 |
| 60003 | 游资数据暂无记录 | 游资行为分析模块 |
| 60004 | 大单阈值设置不合理 | 大单异动分析模块 |

## 五、 接口设计补充说明
1.  **安全性要求**：
   - 敏感接口（如风险规则配置、账户风险查询）需采用 HTTPS 加密传输
   - Token 有效期建议设为 2 小时，支持刷新令牌（refreshToken）机制
   - 对账户相关数据进行脱敏返回，避免完整信息泄露
2.  **性能要求**：
   - 实时接口（如大单实时数据、筹码实时更新）响应时间≤300ms
   - 非实时接口（如历史舆情、技术指标趋势）响应时间≤1000ms
   - 支持接口数据缓存（Redis），缓存过期时间按数据实时性调整（如实时数据缓存5s，历史数据缓存1小时）
3.  **兼容性要求**：
   - 接口返回字段避免使用特殊字符，字段名采用驼峰命名法
   - 新增字段时保持向后兼容，不删除原有核心字段，不修改原有字段类型
4.  **扩展要求**：
   - 预留自定义指标、自定义舆情标签等扩展字段（如 `extParams` 字段）
   - 支持接口批量调用（可选），减少客户端请求次数

### 11.1 数据安全

- 数据传输采用 HTTPS 加密

- 定期进行数据备份与恢复测试

### 11.2 访问安全

- 采用 OAuth 2.0 认证机制
- 实现基于角色的访问控制（RBAC）
- 接口调用频率限制

### 11.3 系统安全

- 定期进行安全漏洞扫描
- 实现入侵检测与防御系统
- 建立安全事件响应机制

## 十二、 性能优化

### 12.1 缓存策略

- 采用 Redis 作为缓存服务器
- 实时数据缓存 5 秒
- 历史数据缓存 1 小时
- 热点数据永久缓存

### 12.2 数据库优化

- 合理设计数据库索引
- 采用分库分表策略
- 实现读写分离

### 12.3 代码优化
