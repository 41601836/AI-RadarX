### 3.2 六大核心模块设计

基于 `项目完整架构汇总报告.md`，AI-RadarX 设计了六大分析模块：

| 模块 | API路径 | 功能描述 |
|------|---------|---------|
| **筹码分析 (Chip)** | `/api/v1/chip/*` | 筹码分布、集中度、支撑阻力位 |
| **舆情分析 (Public)** | `/api/v1/public/*` | 新闻情绪、舆情摘要、热点追踪 |
| **风险控制 (Risk)** | `/api/v1/risk/*` | VaR计算、止损规则、账户风险评估 |
| **热流分析 (Heat)** | `/api/v1/heat/*` | 主力资金流向、龙虎榜数据 |
| **大单分析 (Order)** | `/api/v1/order/*` | 大额订单追踪、机构动向 |
| **技术分析 (Tech)** | `/api/v1/tech/*` | 技术指标、K线形态识别 |

## 五、建议的开发路径

### Phase 1: 独立运行 AI-RadarX (当前阶段)
- 使用 `AI-RadarX/backend` 简化后端
- 依赖 AkShare 获取基础行情数据
- 实现基本的行情展示、搜索、详情功能
- 使用 localStorage 管理用户持仓

### Phase 2: 集成 AI-THink 核心能力
- 配置 AI-RadarX 调用 AI-THink API
- 获取 AI 分析报告、多智能体决策结果
- 展示 LLM 生成的投资建议

### Phase 3: 完整功能融合
- 打通 Tushare 历史数据
- 实现筹码分析、风险控制等高级功能
- 接入 MongoDB/Redis 持久化

## 六、结论

| 项目 | 定位 | 当前状态 |
|------|------|---------|
| **AI-THink** | 核心分析引擎、多智能体系统、LLM集成 | 功能完备，可独立运行 |
| **AI-RadarX** | 前端交易终端UI、用户交互界面 | 框架完成，功能开发中 |

**推荐策略**: 
1. 先完善 AI-RadarX 的前端UI功能（行情中心、投资组合）
2. 再逐步接入 AI-THink 的深度分析能力
3. 最终形成完整的AI辅助交易系统
