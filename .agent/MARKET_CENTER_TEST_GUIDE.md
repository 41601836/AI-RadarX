# AI-RadarX 行情中心测试指南

## 测试环境
- **前端**: http://localhost:3000
- **后端**: http://localhost:8080
- **版本**: v3.2 简化版

## 测试步骤

### 1. 基础功能测试
1. 打开浏览器访问 `http://localhost:3000`
2. 点击页面中央的 **"ENTER TERMINAL"** 按钮
3. 应该看到行情中心页面，包含：
   - 左侧：股票列表（最多显示10只）
   - 右侧：欢迎页面或股票详情

### 2. 数据加载测试
- [ ] 左侧股票列表是否显示了股票名称、代码、涨跌幅？
- [ ] 涨跌幅颜色是否正确（红涨绿跌）？
- [ ] 是否显示"加载中..."（如果数据未到）？

### 3. 交互功能测试
1. **点击股票**：
   - [ ] 点击左侧任意股票
   - [ ] 右侧是否切换到该股票的详情页面？
   - [ ] 被选中的股票是否高亮显示（蓝色边框）？

2. **股票详情页面**：
   - [ ] 显示股票名称和代码
   - [ ] 显示当前价格
   - [ ] 显示涨跌幅
   - [ ] 显示四个分析卡片：
     - 技术分析
     - 筹码分布
     - AI评分
     - 系统建议（显示"强力买入"和置信度）

### 4. 实时更新测试
- [ ] 等待 5-10 秒，观察股票价格是否有变化
- [ ] 涨跌幅数据是否实时更新

### 5. 后端数据源测试
打开浏览器开发者工具（F12），查看 Network 标签：
- [ ] 是否有对 `http://localhost:8080/api/v1/market/quote` 的请求？
- [ ] 请求是否返回 200 状态码？
- [ ] 响应数据中是否包含 `source` 字段（tushare 或 akshare）？

## 预期结果

### 正常情况
- 页面流畅加载，无错误提示
- 股票列表显示真实数据（来自 Tushare 或 AkShare）
- 点击交互响应迅速
- 数据每 5 秒自动刷新

### 可能的问题

#### 问题1: 股票列表为空或显示"加载中..."
**原因**: 后端未启动或数据获取失败
**解决**: 
```bash
# 检查后端状态
curl http://localhost:8080/api/v1/health
```

#### 问题2: 点击股票无反应
**原因**: StockContext 未正确初始化
**解决**: 刷新页面重试

#### 问题3: 数据不更新
**原因**: 实时更新未启动
**解决**: 检查浏览器控制台是否有错误信息

## 数据来源说明

当前配置的数据优先级：
1. **Tushare Pro** (优先) - 需要有效的 API Token
2. **AkShare** (备用) - 免费，无需配置
3. **Mock Data** (最后降级) - 15只模拟股票

查看当前使用的数据源：
- 打开开发者工具
- 查看 `/api/v1/market/quote` 响应
- 检查 `source` 字段值
